Day1

今天老师主要讲了Serverless云开发平台的使用入门。
这个Serverless可以非常方便地把一个团队有效整合在一起，完成开发。任意一个阿里云账号可以创建新的团队，邀请其他人加入团队，可以很方便地队员的权限以及角色进行分配。
创建新的团队
打开https://workbench.aliyun.com/，开箱即用，直接点击右上角的登陆，登陆自己的阿里云账号。点击免费云开发。
登陆之后就可以看到一个空的应用列表，此时还有没项目，或者叫应用。最左上角是团队协同的各个工具。

接着左侧的下拉框可以选择所在的团队或者创建一个新的团队，无缝切换非常方便。
创建一个新的团队，输入团队名，设置好开发人员规模。

创建新的应用
现在部署一个简单的云上HelloWorld程序。可以在「应用列表」中创建新的应用，选择 「空应用」- 「函数计算FC」 - 选择开发语言。这里我选了Java。
接下来依次填入应用的相关信息，也可以为应用指定一个代码仓库（需要事先创建好repo）。
这是创建好之后的应用卡片。

下面，邀请他人加入团队进行协同开发。

添加成员
成员可以是阿里云子账号也可以是主账号。在页面左侧「成员列表」中添加成员。
要想添加主账号，可以把邀请链接发给你想邀请的人，他打开之后选择他自己的主账号登陆即可。
也可以添加子账号。用自己的主账号创建N个子账号，点击「添加阿里云子账号成员」 - 「创建RAM子账号」。
设置子账号的登录名，显示名，以及相关配置。
创建好之后，子账号可以自己使用，也可以发给团队的其他开发者，让他们以子账号登陆我们的应用。
回到「应用列表」，选择「成员管理」，将刚才创建的子账号登录名添加到「开发人员」中。此时我们注意到系统默认为子账号分配了「应用开发者」权限。
子账号登陆之后显示的界面是这样的：

此时，在应用卡片上仅有「进入开发」入口。

进入开发
用子账号进入应用开发。一个炫酷的云IDE被加载出来，这玩意就是今天的主场。它这个IDE应该用的是用开源的VSCODE，可以非常方便地进行项目配置。
加载完之后系统默认导入了 HelloWorld 的项目文件，README中有项目的相关信息，以及怎么跑起来啊之类的。
在 src/main/java/com/alibaba/serverless/helloworld 目录下创建 helloworld.java。
把README中的代码拷贝到 helloworld.java 中。
可以发现，这个 helloword 类只有两个方法，其中， handleRequest 处理请求，返回设置好的内容，主要包括一个 String 类的对象 你好，世界！、一个 Header，另外还有一些比如返回状态啊，加密啥的。

部署
在「部署」标签中，填入路由、函数映射表，com.alibaba.serverless.helloworld.helloworld::handleRequest，默认的文件是 MainHandler ，改成 helloworld，这是我们 handleRequest 所在的文件。保存到文件。点击下方「部署」。
完成之后，根据输出信息访问链接就可以看到函数输出的内容啦~！

今天的学习到此为止，体验了一下云端开发的感觉。期待Day2。
